# Mybatis

Mybatis, 一个持久层的半 ORM 框架，提供了一级缓存、二级缓存、事务管理、多表级联查询等高级特性。

# 1.MyBatis 的使用

在使用 mybatis 时分为注解和 xml 两种方式，且都可以实现相关的业务，但还是推荐使用 XML 方式（ 实现了 SQL 语句与 Java 代码分离，使得代码的可读性更高，而且在写相关的业务逻辑时更加清晰）

在使用 XML 方式时，XML 一般放在在 resource 下。但要注意的一点是，要保正打包后的 XMl 文件和 Mapper 层定义的接口在同一级目录下，所以要根据 mapper 目录所在的层级与 xml 目录的层级一致。

除此之外，在使用 Mybtais 之前还要先配置一个数据源和事务管理器等等。

# 1.1. 关系映射

除开前置要配置的繁杂步骤，使用 xml 方式时最关键的一点是要建立关系映射。其中，关系映射有分为手动映射和自动映射两种方式。

# 1.1.1. 输出关系映射

自动映射

自动映射的方式适用于返回一些简单的数据类型，比如查询的结果是 Java 中的基本数据类型及其包装类时，亦或者是 entity 层中的数据对象，SQL 查询的数据名可以一一映射到此对象的成员变量时（在配置时可以加加上驼峰和下划线之间的转换）。

手动映射

mybatis 中的手动映射通常要自定义一个 resultMap，然后一一为查询结果的字段名与对象实体的成员建立数据映射。其中，根据数据实体的关系又分为三种类型，即一对一、一对多、多对多，但实现的思路是一致的。

一对一： 采用 association 标签建立映射

一对多： 采用 collection 标签建立映射

** 多对多： ** 拆分成两个一对多

# 1.1.2. 输入关系映射

输入关系映射一般通过 paramType 属性设置，且在一般情况下可以忽略（类似于自动映射），不过当遇到比较复杂的参需要自己手动建立映射，此时就要借助动态 SQL 标签。
