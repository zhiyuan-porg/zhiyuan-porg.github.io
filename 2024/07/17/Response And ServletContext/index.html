# 1.Http 响应概述

# 1.1HTTP 响应信息的组成

响应行、响应头、响应体

# 1.2 响应行

由协议及版本、响应状态码、状态信息

[!TIP]

e.g. HTTP/1.1 200 ok

# 1.2.1 响应状态码分为五类

状态码 含义
1xx 指示信息；表示请求已接收，继续处理
2xx 成功；
3xx 重定向
4xx 客户端错误；语法错误 or 无法实现
5xx 服务端错误

# 1.2.2 响应头

由各种键值组成，键固定值可变。常见的响应头如下：

响应头信息 说明
Location 重定向要跳转的地址
Content-Type 设置数据类型及字符集
Refresh 设置稍后跳转的延迟时间及页面路径
Content-Disposition 设置内容的处理方式

[!NOTE]

常见的状态码

200：正确得到响应数据

302：页面重定向

304：使用了本地缓存

400：请求参数错误

404：资源不存在

405：请求的方法错误

415： 无法处理的媒体格式

500： 服务器内部错误

# 1.2.3 响应体

封装服务器要返回的数据，而且可以是任意数据

> [!CAUTION]
    >
    > Get 请求虽然没有请求体，但它有响应体

# 3. 请求相关的方法

# 3.1 作用域的介绍

​ 作用域是不同的 Servlet 之间的数据共享区，存放于数据库的内存中。

​ 主要分为四个：页面域（Page）、请求域（Request）、会话域（Session）、上下文域（ServletContext）

[!WARNING]

数据只能在同一个请求中共享，请求结束或其它的请求不能访问其中的数据。

# 4.1 转发

​ 由服务器进行的页面跳转

# 4.1.1 相关的 API

# 4.1.2 特点

  • 服务器的内部行为，URL 始终不变
  • 客户端只发送了一次请求
  • 请求域的数据不会失效
  • URI 不需要添上项目名

# 4.2 重定向

​ 客户端根据服务器反馈的信息再次请求服务器

# 4.2.1 相关的 API

# 4.2.2 特点

  • 由于是再次请求，所以地址栏会发生变化
